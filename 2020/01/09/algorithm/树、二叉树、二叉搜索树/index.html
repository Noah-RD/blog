算法：树、二叉树、二叉搜索树

发表于 2020-01-09 | 分类于 算法

每种数据结构都有其使用场景下的优点与缺点，而这也是数据结构不断发展与优化的原因之一。

一种思想：升维、空间换时间

树、二叉树、图

简述

在一维的数据结构中，链表中每个结点都指向下一个结点，这些结点线性的串在一起，形成链表这种一维结构。那如果每个节点所指向的不是下一个节点，而是下多个节点呢？这就形成了树。

树是一种二维的数据结构，是由根结点和若干颗子树构成的，不包含子树的结点为叶结点。

从图中可以看出，这是一颗二叉树，具有一个根节点(Root A),下一层分为左右，左边的称为左结点，右边的称为右结点。其中每棵子树都有其父节点、子节点、兄弟结点等。之所以是二叉树，是因为只分叉为左右两颗子树，若分叉为多棵子树，则是 N叉树。

从图中还可以看到，整棵树从上到下没有结点连接的地方，如果出现结点连接而呈现环状，则从定义上称为图(Graph)。

从图中可看到在左边部分，由于结点的连接而出现了环状。综上所述可以认为：链表是特殊化的树，树是特殊化的图。

二叉树遍历

定义一棵二叉树

二叉树的遍历方式包括三种：

  • 前序遍历(Pre-order) : 根->左->右
  • 中序遍历(In-order) : 左->根->右
  • 后续遍历(Post-order) : 左->右->根

举例：

前序顺序： A -> B -> D -> H -> I -> E -> J -> C -> F -> G

中序顺序： H -> D -> I -> B -> J -> E -> A -> F -> C -> G

后续顺序： H -> I -> D -> J -> E -> B -> F -> G -> C -> A

由于二叉树中结点的值是无序的，所以其遍历的时间复杂度是线性的 O(n)，也就是将所有节点遍历一遍。对于树遍历最常用的方法是递归。

搜索二叉树

简述

二叉搜索树(Binary Search Tree)，也称有序二叉树(Ordered Binary Tree)、排序二叉树(Sorted Binary Tree)、是指一颗空树或者具有以下性质的二叉树：

  • 左子树上的所有节点的值均小于它的根结点的值
  • 右子树上的所有节点的值均大于它的根结点的值
  • 以此类推，左、右子树也分别为二叉查找树。(这就是重复性！)

由此可以得出：二叉搜索树的中序遍历为升序。
