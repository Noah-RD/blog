算法：哈希表、映射、集合

发表于 2020-01-08 | 分类于 算法

每种数据结构都有其使用场景下的优点与缺点，而这也是数据结构不断发展与优化的原因之一。

一种思想：升维、空间换时间

简述

因为集合 Set 是一种抽象数据结构，其复杂度根据其底层使用的数据结构而定，如果是 TreeSet 则底层使用的是红黑树(是一种平衡二叉搜索树)。

Hash Table

哈希表(Hash Table) 也叫散列表，是根据关键码值(key value)而直接进行访问的数据结构。它通过把关键码值映射(通常使用映射函数)到表(哈希表，通常使用数组实现)中的一个位置来访问记录，以加快查找的速度。而这个映射函数叫做散列函数(Hash Function)，存放记录的数组叫做哈希表(或散列表)。

通俗讲，就是当存储或访问一个键所对应的值得时候，先通过哈希函数将键 hash 为一个整数值作为数组中的一个下标，然后将值放到此下标数组中或返回即可。

这里最关键的就是哈希函数，如果哈希函数实现的不好，则会发生多个键 hash 成相同的整数值，也就是相同的数组下标，这种情况称为哈希碰撞。

为了处理这种碰撞，于是在数组每个单元中存储的是一个链表(link list)，当键 hash 值相同时，便对此数组单元的链表进行操作，查询则遍历链表，插入则加到链表最后。

正是因为这种基于 hash 的操作，使得哈希表的查询、插入、删除皆为常数级时间复杂度 O(1)。

但有一种极端情况下，哈希表的效率会发生退化，就是当哈希函数很差，导致数组单元内链表特别长，或者大部分数据都集中在一个链表中，这时，哈希表变退化为了链表这种数据结构。这将导致其查询、插入、删除的时间复杂度皆退化为 O(n)。

所以比如 JAVA 的 HashMap 中，当 hash 冲突到一定程度的时候，会把链表转为红黑树的优化处理。

Set

集合(Set) 是一个抽象的数据结构，其不同的实现会具有不同的应用场景与时间复杂度。比如 HashSet(由哈希表作为底层实现)、TreeSet(由红黑树作为底层实现)等。
