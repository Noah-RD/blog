算法：数组、链表、跳表

发表于 2020-01-07 | 分类于 数据结构与算法

每种数据结构都有其使用场景下的优点与缺点，而这也是数据结构不断发展与优化的原因之一。

一种思想：升维、空间换时间

简述

数组 是数据结构中最常见的一种，也是很多其他数据结构实现的基础，其随机访问(下标访问)复杂度为常数级的 O(1)。但是插入与删除操作却效率很低，是线性级的 O(n)。于是为了提高插入与删除操作，便出现了链表。

链表 的插入与删除操作都是常数级的 O(1)，这弥补了数组的线性复杂度，但链表的访问复杂度却是线性级的 O(n)，那怎么才能继续优化访问的复杂度呢？跳表随之而来。

跳表 最常见的场景便是 redis 中。其设计的思想是：一维转二维，空间换时间的思想。 其访问、搜索、插入、删除的时间复杂度皆为 O(log(n))，但是空间复杂度为 O(nlog(n))，而数组与链表的空间复杂度为 O(n)。

正如开头所说，每种数据结构都有其优缺点，其取舍取决于其使用场景。

数组

数组在底层硬件实现中，每当你申请数组时，会在内存中开辟一段连续的内存地址，每一个地址可以通过内存管理器直接进行访问。

也真是由于数组采用顺序存储的方式，使其访问复杂度为常数级 O(1)，但也因此，插入与删除为线性级O(n)。

(数组在内存中示意图)

访问操作： 由于内存地址连续，所以数组的随机访问(下标访问)的时间复杂度为 O(1)

搜索操作： 比如搜索数组中(乱序)等于几的值，需要从头遍历数组，所以复杂度为 O(n)

插入操作： 插入前，需先挪动插入点之后的所有元素依次后移一位，再进行插入操作，所以复杂度为 O(n)

删除操作： 删除后，将删除点后所有元素依次向前挪动一位(与插入操作相反)，所以复杂度为 O(n)

有很多数据结构是以数组为基础构建的，比如数组链表()、哈希表(HashMap) 等。数组这种数据结构非常稳定，其复杂度不会因为极端情况而退化。

链表

链表由一系列结点 Node 组成(第一个节点为头结点 Head ，最后一个是尾结点 Tail )，每个节点包括一个数据域 value 和指向下一个节点的指针域，结点数量可在运行时动态生成。

其物理存储单元是非连续的、非顺序的存储结构，其逻辑顺序通过结点中的指针链接次序实现的(即一个结点指向另一个结点)。

链表包括：单向链表(结点中只有 next 指针指向下一个结点，最后一个结点 next 指向空)、双向链表(结点中既有 next 指向下一个节点，又有 prev 指向前一个节点)和循环链表(尾指针指向头指针)。

一个最简单的 LinkedList 定义(JAVA)：

访问与搜索操作： 由于链表的存储单元是非连续、顺序的，所以其访问或搜索只能从头结点向后依次遍历，或从尾结点向前依次遍历，所以其时间复杂度为 O(n)

插入与删除操作： 也正是由于链表的存储方式，使其插入或删除无需挪动其他结点，只需修改相应位置的结点指针即可，所以其时间复杂度为 O(1)

从上图可以看出，链表的插入与删除(与插入原理相同)操作仅进行了修改指针的操作，所以时间复杂度为常数级的。链表的数据结构也很稳点，不会在特殊情况下而退化。

虽然链表这种数据结构将数组低效率的插入与删除，从线性复杂度提升到了常数级，但其查询效率却变为了 O(n) 的，那怎么给链表加速才能将读写的效率平衡呢？于是有了跳表。

跳表

数组和链表是一维的数据结构，为了弥补一维结构的一些不足，于是便有了升维的思想。

跳表是一个随机化的数据结构，可以被看做二叉树的一个变种，其实质就是一种可以进行二分查找的有序链表。跳表在原有的有序链表上面增加了多级索引，通过索引来实现快速查找。

可以看到，跳表在原有的有序链表上面增加了多级索引，通过索引来实现快速查找。首先在最高级索引上查找最后一个小于当前查找元素的位置，然后再跳到次高级索引继续查找，直到跳到最底层为止，这时候已经十分接近要查找的元素的位置了(如果查找元素存在的话)。由于根据索引可以一次跳过多个元素，所以跳查找的查找速度也就变快了。

比如上面的图要查询 8 这个数，在二级索引找到小于 8 的元素 7，于是从 1 跳到 7。
接着从 7 来到一级索引发现小于 8 的还是 7，于是来到原始链表，找到 8。可以看到相比于原来的从 1 遍历到 8，跳表只需要 3 次即可。

当链表中数据较多时，只要相应的增加索引级即可，那具体增加多少级索引呢，log2(n) 个。从这里也体现了一种思想即空间换时间

时间复杂度

跳表的时间复杂度是怎样的呢？从上图可以看出，原始链表假设有 N 个结点，一级索引为 N/2，二级索引为 N/4 以此类推：

N/2、N/4、N/8、第 k 级索引结点的个数就是 N/(2^k)。

那么假设索引一共有 H 级，最高级索引有 2 个结点。即表示为：N/(2^H) = 2，从而求得 H = log2(n) - 1。所以跳表的查询复杂度为 O(logn) 的。

比如如果原始链表有 1024 个结点，那假如查找最后一个结点则需要遍历 1024 次，而用跳表只需要 10 次 (log2(1024)) 即可。

现实中使用

在现实中，由于跳表会有插入与删除的操作，这会使其索引结构不那么工整，比如有的索引可能会多跨几步，而有的则少跨几步。所以对于插入与删除操作来说，每次操作后都要进行索引的调整，所以其插入与删除操作的复杂度为对数级 O(logn) 的。

从图中可以看出，跳表主要由以下几部分构成：

  • 表头（head）：负责维护跳跃表的节点指针。
  • 跳跃表节点：保存着元素值，以及多个层。
  • 层：保存着指向其他元素的指针。高层的指针越过的元素数量大于等于低层的指针，为了提高查找的效率，程序总是从高层先开始访问，然后随着元素值范围的缩小，慢慢降低层次。
  • 表尾：全部由 NULL 组成，表示跳跃表的末尾。
