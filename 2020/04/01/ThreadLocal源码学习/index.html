ThreadLocal源码学习

发表于 2020-04-01 | 分类于 JAVA源码

每个线程都有自己专有的，用来存放数据的 ThreadLocalMap。而 ThreadLocal 就是用来操作这个 Map 的工具类。

ThreadLocal 并不实际存储数据，只是提供了一组方法，用来操作属于当前线程自己的一些变量。

ThreadLocal 是如何实现的线程资源隔离的呢？还是要用到 Thread 线程类本身，每个 Thread 线程都有一个属于自己的 ThreadLocalMap，所有的本线程使用的变量，都放到这个属于自己的 Map 中，也就实现了隔离的效果。
而 ThreadLocal 只是提供一些通用方法来帮助 Thread 管理这个 Map 而已，以及 ThreadLocalMap 的实现代码在 ThreadLocal 中。

我们先来看一下这个 Map

它是 Thread 的静态内部类，是实际用来保存当前线程数据的容器。这个 Map 是专门为 ThreadLocal 单独实现的 Map，其内部用来存储数据的是一个 Entry[]。
数组中每个 Entry 有一个key 和 value，value 就是要存储的数据，而 key 是一个 ThreadLocal 的弱引用。也就是说，在这个 Map 中，是通过 ThreadLocal 作为 key 来进行查询的。而如何进行查询，如何设置值，如何删除这些通用操作，就是由 ThreadLocal 进行了定义。
