PriorityQueue

发表于 2021-03-31 | 分类于 集合

PriorityQueue 是一个基于堆数据结构的无界队列。

概览

一个基于优先堆的无界优先队列。队列中元素的顺序根据其自然排序，或通过构造器显式指定的 Comparator 排序。优先队列不允许空元素。如果一个优先队列中元素的排序依赖其自然顺序，则所有的元素都必须是可比较的，如果插入不可比较的元素，可能会抛出 ClassCastException。

就指定的顺序而言，队列的头元素是最小的元素。如果多个元素同时都是最小的元素，则头元素是它们中的一个 – 关系被任意打破。poll、remove、peek、element 等访问操作，在队列头部进行。

优先队列是无界的，但是基于其内部是由数组实现的，所以有一个默认的最大容量限制。它总是至少与队列大小一样大。当添加新元素时，其容量会自动增长。并没有指定具体的容量增长策略。

其实现了所有 Collection 和 Iterator 接口中的可选方法。其迭代器的迭代顺序是不确定的，如果你需要确保迭代顺序，请考虑使用 Arrays.sort(pq.toArray()).

注意，PriorityQueue 不是线程安全的。如果线程具有修改 PriorityQueue 的操作时，PriorityQueue 不应该被多个线程并发访问。如果需要并发安全，可以使用 PriorityBlockingQueue 替代。

实现提示：进队与出队操作(offer、poll、remove()、add)等操作的时间复杂度为 O(log(n))。remove(Object) 与 contains(Object) 方法的时间复杂度为 O(n)，也就是线性的。检索操作如 peek、element、size 等时间复杂度是 O(1) 的，也就是常量。

这个类是 java 集合框架的一员。

数据结构

想学习 PriorityQueue，首先要知道什么是优先堆。优先堆是一种排序算法，按照排序的升序与降序分为小顶堆与大顶堆。

那什么是优先堆呢？其实就是一颗完全二叉树，小顶堆就是每棵子树的根结点值，小于其左右子结点。大顶堆则相反。我们来看一下什么是完全二叉树：

完全二叉树，就是除了叶子结点以外，都是满二叉树。所有的叶子结点也堆满在左侧。当然了，叶子结点也满了，那就是满二叉树了。这种数据结构就是优先堆，其根结点的值，就是最小值或最大值。

那我们如何模拟完全二叉树这个数据结构呢？很简答，数组。将树的所有结点，依次按照每一层，从上到下，从左向右分别放入数组中。然后我们可以有一个数组索引和树结点的对应关系：

PriorityQueue 在堆排中，用到了两个方向的堆化操作，上面的排序例子中，其实只有从上往下的构建堆化(找到一个父节点，然后递归堆化后续的子树)。而 PriorityQueue 还有一个从下向上的堆化，也就是找到一个子结点，然后不断往上比较父结点。两个道理都是一样的。

PriorityQueue 的自下而上和自上而下的两种堆化方式，就可以从任意一个结点开始，构建优先堆了。

队列操作

关于队列的操作，一共看三个就行，offer、poll、remove。其他的都很简单。这三个方法，基础逻辑都很简单，所有的核心逻辑都在 siftUp 与 siftDown 两个堆排方法中。

这两个方法，分别根据是否指定比较器，而分出了两个方法，但除了使用啥比较的语句不同外，其他的都一样，所以我们分别看一个就行了。

可以看到，源码中的堆化使用的是迭代而不是递归(大神就是大神啊)。核心逻辑就是判断父结点与两个子结点的大小，然后换位置，然后在往下继续。

以上就是 PriorityQueue 两个核心的堆化算法。然后我们一起看一下那三个方法

offer 方法为什么要使用 siftUp 从下往上堆化呢？因为参数 i 是数组的最后一个元素，所以得往上了。

堆算法，并不是一次性排好序的，而是在每一次 poll 的时候，重新用堆化算法找一遍。也就是，数组中的元素其实一直是乱序的，并不是已经排好序的。在每一次插入和获取操作的时候，都进行一次最大/最小值的查找。

删除操作，最后用到的是 removeAt(int i) 方法，这个方法可以删除堆中，任何一个位置的元素，所以当位置任意的时候，我们就得同时使用 siftUp 和 siftDown 来保证堆的特性。

到此位置，PriorityQueue 就差不多了，其核心是要理解堆排算法。
