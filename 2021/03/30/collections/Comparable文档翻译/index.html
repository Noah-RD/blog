Comparable文档翻译

发表于 2021-03-30 | 分类于 集合

文本为对 JAVA SE 8 java.lang 包下 Interface Comparable<T> 文档的翻译。

接口概述

此接口对实现它的每个类的对象施加总顺序。这种排序称为类的自然排序，而类的 compareTo 方法称为其自然比较方法。

实现此接口的对象所在的 List(和 Array)可以被 Collections.sort(和 Arrays.sort) 自动排序。当 sorted map 中的 key 或 sorted set 中的元素对象实现此接口后，则无需在指定提供一个比较器(comparator)。

对于类 C 的每个 e1 和 e2 对象,当且仅当 e1.compareTo(e2) ==0 具有与 e1.equals(e2) 相同的布尔值时，我们才称类 C 的自然顺序与 equals 一致。请注意，null 不是任何类的实例，即使 e.equals(null) 方法返回 false，e.compareTo(null) 也应该抛出 NullPointerException。

强烈建议(尽管不是必需的)使自然顺序与 equals 行为保持一致。这是因为当使用没有显示指定比较器的 sorted sets 或 sorted maps 时，如果其中的元素 或 key 的自然顺序与 equals 行为不一致时，会引发奇怪的表现。特别是当这样的 sorted set(或 sorted map) 违反了根据 equals 方法定义的 set(或map) 的一般约定。

例如，如果将两个具有 (!a.equals(b) && a.compareTo(b) == 0) 关系的对象，作为键放入没有指定显示比较器的 sorted set 时，在添加 a 后在添加 b 则返回 false(并且 sorted set 的大小不会改变)，因为从 sorted set 的角度看，a 和 b 是等价的。

几乎所有实现 Comparable 的 Java 核心类都具有与 equals 一致的自然顺序。一个例外是 java.math.BigDecimal 类，其自然顺序等同于具有相同值和不同精度的 BigDecimal 对象(例如 4.0 和 4.00)。

方法概览

compareTo

int compareTo(T o)

将此对象与指定的对象进行比较。当此对象小于、等于或大于指定对象时，返回负整数、零或正整数。

实现者必须确保所有 x 和 y 的自反性： sgn(x.compareTo(y)) == -sgn(y.compareTo(x))(这意味着 x.compareTo(y) 必须在 y.compareTo(x) 抛出异常时抛出异常。)

实现者还必须确保关系的传递性：(x.compareTo(y) > 0 && y.compareTo(z) > 0) 意味着 x.compareTo(z) > 0。

最后，实现者必须确保对等性：对于所有 z，x.compareTo(y)==0 意味着 sgn(x.compareTo(z)) == sgn(y.compareTo(z))。

强烈建议，但不是严格要求 (x.compareTo(y) == 0) == (x.equals(y))。一般来说，任何实现可比较接口并违反此条件的类都应该清楚地指出这一事实。推荐的语言是 “注意：这个类有一个与 equals 不一致的自然顺序。”(Note: this class has a natural ordering that is inconsistent with equals.)

在前面的描述中，符号 sgn(表达式) 指定数学符号函数，该函数被定义为根据表达式的值是负的、零的还是正的来返回 -1、0 或 1 中的一个。

返回：负数(小于)、0(等于)、正数(大于)
