Collection文档翻译

发表于 2021-03-23 | 分类于 集合

文本为对 JAVA SE 8 java.util 包下 Interface Collection<E> 文档的翻译。

接口概述

集合层次结构中的根接口。集合表示一组对象，称为其元素。有些集合允许重复元素，有些则不允许。有些是有序的，有些是无序的。JDK 不提供该接口的任何直接实现: 它提供了更具体的子接口的实现，如 Set 和 List。这个接口通常用于在需要最大通用性的地方传递和操作集合。

Bags 或 multisets(可能包含重复元素的无序集合)应该直接实现此接口。

所有通用 Collection 实现类(通常通过其子接口之一间接实现 Collection)都应该提供两个“标准”构造函数: 一个 void (无参数)构造函数，它创建一个空集合; 另一个构造函数有一个 Collection 类型的参数，它创建一个新集合，其元素与它的参数相同。实际上，后一个构造函数允许用户复制任何集合，生成所需实现类型的等效集合。没有办法强制执行这个约定(因为接口不能包含构造函数) ，但是 Java 平台库中的所有通用的 Collection 实现都遵守这个约定。

此接口中包含的“破坏性”方法(即修改其操作所在的集合的方法)被指定在该集合不支持该操作时抛出 UnsupportedOperationException。如果是这种情况，那么如果调用对集合没有影响，那么这些方法可能会抛出 UnsupportedOperationException，但不是必需的。例如，对不可修改的集合调用 addAll (Collection)方法时，如果要添加的集合为空，则可以抛出异常，但不要求这样做。

有些集合实现对它们可能包含的元素有限制。例如，有些实现禁止空元素，有些对其元素的类型有限制。试图添加不合格的元素会引发未检查的异常，通常是 NullPointerException 或 ClassCastException。尝试查询不合格元素的存在可能会抛出异常，或者它可能只是返回 false; 有些实现会显示前者的行为，有些会显示后者。更一般地说，如果一个不合格的元素的完成不会导致将不合格的元素插入到集合中，那么尝试对该元素进行操作可能会引发异常，或者在实现的选项中成功。在此接口的规范中，此类异常被标记为“可选”。

由每个集合决定自己的同步策略。在缺乏更强有力的实现保证的情况下，集合上的任何方法被另一个线程变异都可能导致调用未定义行为; 这包括直接调用，将集合传递给可能执行调用的方法，以及使用现有迭代器检查集合。

集合框架接口中的许多方法都是根据 equals 方法定义的。例如，contains (Object o)方法的规范说: “返回 true，当且仅当这个集合至少包含一个元素 e 使得(o = = null？E = = null: o.equals (e))”这个规范不应该被解释为意味着使用非空参数 o 调用 Collection.contains 将导致对任何元素调用 o.equals (e)。实现可以自由地实现优化，从而避免对 equals 的调用，例如，首先比较两个元素的散列代码。(Object.hashCode ()规范保证两个具有不同哈希码的对象不能相等。)更一般地说，只要实现者认为合适，各种集合框架接口的实现都可以自由地利用基础 Object 方法的特定行为。

对集合执行递归遍历的某些集合操作可能会失败，对于集合直接或间接包含本身的自引用实例可能会有例外。这包括 clone ()、 equals ()、 hashCode ()和 toString ()方法。实现可以选择性地处理自引用场景，但是大多数当前的实现不这样做。

这个接口是 Java 集合框架的一个成员。

实现要求: 默认的方法实现(继承的或其他的)不应用任何同步协议。如果 Collection 实现具有特定的同步协议，则它必须重写缺省实现以应用该协议。

方法概览

size

int size()

返回当前集合元素的个数。如果当前集合的元素数量超过 Integer.MAX_VALUE，则返回 Integer.MAX_VALUE。

返回：当前集合元素的个数

isEmpty

boolean isEmpty()

如果当前集合不包含任何元素，则返回 true。

返回：不包含任何元素，则返回 true。

contains

boolean contains(Object o)

如果当前集合包含指定元素，则返回 true。更通俗的讲，当且仅当，当前队列至少包含一个元素满足 (o==null ? e==null : o.equals(e))时，返回 true。

返回：如果包含指定元素，则返回 true。

iterator

Iterator<E> iterator()

返回此集合元素的迭代器，不能保证元素返回的顺序(除非当前集合所属类，提供此保证)。

返回：一个当前集合元素的迭代器。

toArray

Object[] toArray()

返回一个包含当前集合所有元素的数组。如果当前集合可以保证通过迭代器返回的有序性，则此数组也按照这个顺序返回。

返回的数组是安全的，当前集合不存在任何指向该数据的内部引用(换句话说，这个方法必须重新分配一个新的数组，即使当前队列的内部数据结构也是一个数组)。因此，对于调用者而言，可以放心的去修改数组。

这个方式，扮演着基于数组与基于集合的 API 桥梁。

返回：包含当前集合所有元素的数组。

toArray

<T> T[] toArray(T[] a)

返回一个包含当前集合所有元素的数组。返回数组的运行时类型由参数指定。如果当前集合适合指定的数组，则直接返回包含所有元素的数组。否则，将使用指定数组的运行时类型和此集合的大小分配一个新数组。

如果此集合适合指定的数组并有剩余空间(即，数组比该集合具有更多的元素)，则紧接集合结束后的数组中的元素将设置为 null。(仅当调用者知道此集合不包含任何null元素时，此方法才可用于确定此集合的长度。)

如果此集合对由其迭代器返回其元素的顺序进行了任何保证，则此方法必须以相同的顺序返回元素。

与 toArray() 方法一样，此方法充当基于数组的 API 和基于集合的 API 之间的桥梁。此外，此方法允许对输出数组的运行时类型进行精确控制，并且在某些情况下可以用来节省分配成本。

add

boolean add(E e)

确保此集合包含指定的元素(可选操作)。如果此集合由于调用而更改，则返回 true。(如果此集合不允许重复并且已经包含指定的元素，则返回false。)

支持此操作的集合可能会对可以添加到此集合中的元素施加限制。特别是，某些集合将拒绝添加空元素，而其他集合将对可能添加的元素类型施加限制。集合类应在其文档中明确指定对可以添加哪些元素的任何限制。

如果某个集合,除已包含该元素以外的其他任何原因拒绝添加该元素，则它必须引发异常(而不是返回false)。这保留了不变，即在此调用返回之后，集合始终包含指定的元素。

返回：如果此集合由于调用而发生更改，则返回 true。

remove

boolean remove(Object o)

如果存在，则从此集合中删除指定元素的单个实例(可选操作)。更正式地讲，如果此集合包含一个或多个满足 (e == null？e == null：o.equals(e)) 的元素，则删除元素。如果此集合包含指定的元素(或者等效地，如果此集合由于调用而更改)，则返回true。

返回：如果此调用删除了元素，则返回 true。

containsAll

boolean containsAll(Collection<?> c)

如果当前集合包含所有，参数指定集合中的元素时，则返回 true。

addAll

boolean addAll(Collection<? extends E> c)

将指定集合中的所有元素添加到此集合中(可选操作)。如果在操作进行过程中修改了指定的集合，则此操作的行为是不确定的。(这意味着如果指定的集合是此集合，并且此集合是非空的，则此调用的行为是不确定的。)

返回：如果此集合由于调用而发生更改，则为true。

removeAll

boolean removeAll(Collection<?> c)

删除当前集合中，同时包含在当前集合与给定集合中的元素(可选操作)。在此调用返回之后，此集合将不包含与指定集合相同的元素。

返回：如果此集合由于调用而发生更改，则为true。

retainAll

boolean retainAll(Collection<?> c)

仅保留此集合中包含在指定集合中的元素(可选操作)。换句话说，从此集合中删除所有未包含在指定集合中的元素。

返回：如果此集合由于调用而发生更改，则为true。

clear

void clear()

删除当前集合中的所有元素(可选操作)。此方法返回后，当前集合为空。
